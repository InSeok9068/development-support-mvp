   - 2.3 자산금액

   - 2.5 손익금액 (존재한다면)

3. 추출된 자산정보를 바탕으로 괸라자 데이터베이스의 자산정보와 매칭한다.
4. 관리자 자산정보에는
   - 자산카테고리
   - 자산명
   - 자산명 (별명1)
   - 자산명 (별명2)
   - 자산명 (별명3)
   - 자산 구분
     - 현금성(Liquid / Cash-like)
       - 현금, 예금, 적금, MMF(현금성 펀드), CMA 등
       - 목적: “언제든 쓸 돈 / 안전”
     - 투자성(Risk / Growth-oriented)
       - 주식, 주식형 ETF, 주식형 펀드, 암호화폐, 테마 ETF 등
       - 목적: “수익 추구 / 변동 감수”
     - 안정·이자형(Defensive / Interest-oriented)
       - 채권(국채/회사채), 채권형 ETF, 채권형 펀드, 단기채, 머니마켓(현금성이랑 경계)
       - 목적: “변동 낮추고 이자/방어”
     - 실물·대체(Real / Alternative)
       - 부동산, 리츠, 금/원자재, (원하면 자동차도 여기 포함 가능)
       - 목적: “실물자산/인플레 헤지/상관분산”
   - 자산 성향(태그) : 복수선택

     - 3. 혼합

6. 매칭된 자산정보를 바탕으로 사용자에게 응답값을 내려준다.
7. 매칭이 실패된 정보도 기본정보를 포함해서 내려준다.
8. 매칭이 실패한 정보는 관리자가 직접 볼 수 있으면 추가적으로 관리자 페이지에서 확인 및 정보입력이 가능해야한다.
